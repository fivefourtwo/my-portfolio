User Journey map


We documented the results of the discussions  with the consultant and the developer and our  own functional tests and finally analysed them  in a user journey map. To do this, we divided  the user journey map into the categories of  learning to control, testing driving, testing  functions and testing seating position. Kano Model


The Kano model is a valuable framework for  understanding customer preferences and  prioritising features. In the context of our  power wheelchair control system, we can  categorise the different joystick and grip  shape options based on their impact on user  satisfaction.


Some options might fall into the "must-have"  category, such as basic joystick functions and  a standard grip shape. Others might be "onedimensional" features that increase  satisfaction,  satisfaction when they are present, such as a  touch-sensitive joystick or an ergonomic grip  shape. In addition, we can explore "attractive"  features that have the potential to pleasantly  surprise users, such as haptic feedback in the  joystick or a customisable grip shape with  different materials or designs.

Moscow Method


The MoSCoW method helps us to prioritise  requirements by dividing them into the  categories "Must", "Should", "Could" and  "Won't" for the first version of our electric  wheelchair control system.


The "must-have" requirements include a  reliable and responsive control system with  interchangeable joysticks and gripping aids.


Should-have requirements include features  that improve user comfort and ergonomics,  such as adjustable joystick sensitivity and  customisable grip sizes.


Possible requirements could include advanced  features such as voice control or gesture  recognition, while won't-have requirements  would be features that are beyond the scope  of the first version.

Prototype Ⅰ


Through discussions and consultations, we  selected three designs for the production of  clay models. When determining the  dimensions of the model, we tried to take into  account the comfort for people with different  hand sizes. When designing the shape of the  model, we also took into account the findings  of ergonomics to make the model more  comfortable. One of the main considerations  was the allocation of functions to the different  buttons. For example, where the joystick  should be placed for directional control,  whether it would be easier to combine the  horn with the joystick, or whether it would be  more convenient to have a separate indicator  button, like on a car, etc.

Konzeptüberarbeitung


During the tests, however, we realised that no  single model is suitable for all people. That's  why we tried to divide our model into several  parts. If each part can be freely combined, we  can appeal to a larger number of people and  save the budget at the same time. Based on  this concept, we have once again focussed on  all the necessary control functions. After a  review of the Moskow model, we finally  decided to focus only on the functions that  are absolutely necessary (i.e. steering,  indicators, horn and switches).


Die erste Personengruppe:  Nur die Finger können flexibel verwendet werden.  Daher wird ein Steuergerät benötigt, das in einer festen  Position bedient werden kann. Um das Design komfortabler  zu gestalten, haben wir uns bei der Formgebung an der  Ergonomie einer Computermaus orientiert.


In order to fulfil the users' habits, we tried  different control options such as joystick,  slider, buttons, etc. We decided to integrate all  the necessary functions into one main module  that can be combined with other external  models. Different external models are suitable  for different users. In this way, we can create  the most suitable models based on the users'  individual abilities.


Die zweite Personengruppe:  Die gesamte Hand kann flexibel verwendet werden.  Daher liegt der Schwerpunkt unseres Designs darauf, den  Benutzern einen bequemen Handgelenkwinkel zu  ermöglichen. Im Laufe des Designs stellten wir fest, dass es  schwierig war, alle Funktionen in den vorderen Bereich zu  integrieren, weshalb wir den Blinker an die Rückseite  platziert haben.


Die dritte Personengruppe:  Der gesamte Arm kann genutzt werden, aber nicht  präzise.  Deshalb benötigen sie mehr Stützkraft für die Hand und  einen festen Sockel. Wir haben das vorherige Design des  Sockels übernommen und bieten lediglich einen  austauschbaren oberen Teil an. Dies dient auch der  Kostenersparnis beim Design.

Prototype Ⅱ


To check the feasibility of our idea, we  created further test models. Based on  ergonomic principles, we developed two  different external models. One is suitable for  users who can move their entire hand and arm  freely, the other for users who can only move  their fingers freely. In addition, we have  developed different interchangeable control  methods for the directional function to ensure  that the module with all the main functions is  suitable for different users.

Conclusion
